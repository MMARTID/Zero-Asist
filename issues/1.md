## 🐛 Descripción

En `app/main.py` (línea 44) se calcula correctamente el hash SHA-256 del contenido del archivo (`document_hash`), pero **nunca se verifica en Firestore si ya existe un documento con ese hash** antes de proceder al guardado.

Esto provoca que subir el mismo documento dos veces genere dos entradas duplicadas con diferente `doc_id`.

## Código afectado

**`app/main.py` — línea 44 y líneas 99–108**

```python
doc_hash = hashlib.sha256(contenido).hexdigest()
# ⚠️ Nunca se consulta Firestore para verificar si doc_hash ya existe
...
guardar_documento("documentos", doc_id, { ... })
```

## Solución sugerida

Antes de guardar, hacer una query a Firestore buscando si ya existe un documento con el mismo `document_hash`. Si existe, retornar el documento existente (o un error 409 Conflict) en lugar de crear uno nuevo.

```python
# Ejemplo de verificación previa
existing = db.collection("documentos").where("document_hash", "==", doc_hash).limit(1).get()
if existing:
    raise HTTPException(status_code=409, detail="Documento duplicado")
```

## Impacto

- Duplicados en Firestore
- Costes innecesarios de almacenamiento y procesamiento con Gemini
